# Solo Context

For Ayric-led projects that are solo or solo-with-clients (no internal team to coordinate with). The Compex team-roster doesn't apply here — use this instead.

Reference this from: Ayric-Personal, Ayric-Business, Ayric-Learning-MBA, Ayric-Learning-Fabric-AI, Korry-PBI-Audit-and-Improvement.

Last updated: 2026-04-16

---

## Ayric Intelligence (consulting LLC)

### Who's involved
- **Ayric** — sole operator. All client delivery, sales, ops.
- **Clients** — engagement-specific (see Korry section below for current active client).

### Operating assumptions
- No employees, no vendors (today). All work is solo.
- Client-facing outputs need a more polished register than internal Compex voice — still plain English, still concise, but no shorthand or inside references.
- When a project is flagged as an Ayric Intelligence client engagement, the template should prompt: "Is this a client engagement? If yes, apply client voice + capture Voice of the Customer artifacts."

## When roles cross contexts

- If a conversation touches both Compex and an Ayric Intelligence client (e.g., an engagement referral coming through Compex): default to Ayric Intelligence voice for the client-facing portion, Compex voice for the internal portion, and flag the boundary explicitly.
- Never carry confidential Compex team/stakeholder info into Ayric Intelligence deliverables, even as examples.
